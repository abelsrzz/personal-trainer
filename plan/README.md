# Plan Web Product

Esta carpeta recoge el plan de mejora del portal web de RunPilot y de las capacidades operativas que lo alimentan.

## Archivos

- `web_product_improvement_plan.md`: roadmap funcional y tecnico.
- `backlog.yaml`: backlog priorizado para ejecucion.

## Alcance

El plan cubre 10 iniciativas:

1. Replanificacion real desde la web.
2. Check-in diario rapido.
3. Comparativa plan vs realidad.
4. Centro de decisiones del coach.
5. Seguimiento de molestias y riesgo.
6. Semana operativa mas accionable.
7. Insights de progreso reales.
8. Preparacion de carrera.
9. `/chat` en la web conectado al mismo OpenCode remoto que hoy usa Telegram.
10. Nutricion, hidratacion y suplementacion para competiciones y sesiones exigentes.

## Criterio de prioridad

1. Cerrar el bucle entre observacion, decision y accion.
2. Reducir friccion diaria para el atleta.
3. Reutilizar la arquitectura ya existente antes de introducir piezas nuevas.
4. Entregar valor visible con cambios pequenos y verificables.
