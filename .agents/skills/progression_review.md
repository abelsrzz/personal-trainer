# Skill: Progression Review

## Purpose

Detect progression or regression markers when reviewing a completed workout.

## Goal

Do not judge a session only in isolation. Compare it against prior sessions with similar distance, terrain demand and coaching objective to see whether efficiency is improving, stable or worsening.

## Comparison Rules

1. match the same broad goal category
2. prefer similar planned distance
3. prefer similar elevation gain
4. compare recent references first
5. avoid drawing strong conclusions from a single noisy comparison

## Main Progression Markers

- lower heart rate at similar pace
- faster pace at similar heart rate
- better pace with similar or lower power cost
- lower decoupling for similar session type
- more stable splits for the same objective
- ability to absorb the week afterwards without red/yellow fallout
- hybrid support tolerated well enough that bike can continue carrying aerobic or intensity support without masking tibial limits

## Regression Markers

- slower pace with higher heart rate
- higher heart rate without pace benefit
- much worse decoupling than comparable sessions
- loss of control in the second half on sessions that should stay aerobic

## Output Rules

- state whether progression, regression or no clear signal is present
- keep wording probabilistic, not dramatic
- mention the comparable sessions used
- separate fitness signal from contextual noise like heat, terrain or execution errors
- separate metabolic progression from mechanical tolerance: a bike-supported week can improve engine markers without proving that running impact can yet progress at the same rate

## Preferred Automation

The Garmin planned-session reviewer should include this automatically in each generated analysis and markdown review.
