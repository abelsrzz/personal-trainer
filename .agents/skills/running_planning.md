# Skill: Running Planning

## Purpose

Turn long-term goals into blocks and weeks.

## Core Principles

- consistency first
- shin safety first
- use heart rate for easy control
- use pace mainly for quality work
- do not fake threshold precision if threshold is unknown
- treat the 35:00 goal as aspirational until checkpoints support it
- use `planning/goal_gates.yaml` before allowing 35:00-specific prescriptions
- use `planning/coach_decision.md` before changing weekly load
- use `system/state/athlete_state.json` as the primary consolidated read for absorbed week, pace bands, bike-support requirement and blocked progression dimensions

## Weekly Planning Checklist

1. identify active block
2. read latest week and reviews
3. refresh or read `athlete/status_dashboard.md` and `planning/coach_decision.md`
4. inspect recent Garmin or manual activity data
5. assess pain, fatigue, compliance and `athlete/shin_tracker.yaml`
6. read `system/state/athlete_state.json` for progression, hybrid load and pace-band guidance
7. build the next week accordingly
8. if `planning/weeks/semana_actual.md` is created or modified, generate the PDF and send it by Telegram

## Additional Rules

- after impact return starts, assume roughly `+5%` running progression over the last absorbed week unless coach color, pain or fatigue say otherwise
- keep at least one bike support session in most weeks while running durability is still being rebuilt
- use fartlek regularly in early return and base phases as a bridge before denser interval work
- move tempo/VO2 support to bike when the metabolic stimulus is useful but the tibial cost of running it would be too high
