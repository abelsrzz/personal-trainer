# Skill: Athlete Data Maintenance

## Purpose

Keep athlete data structured, current and useful for planning.

## Update Targets

- `athlete/profile.yaml`
- `athlete/health.yaml`
- `athlete/zones.yaml`
- `athlete/shoes.yaml`
- `athlete/preferences.yaml`
- `athlete/shin_tracker.yaml`
- `athlete/status_dashboard.md`
- `athlete/history/`

## Rules

- preserve history
- avoid deleting evidence
- update structured YAML first, then supporting narrative files if useful
- log periosteum pain in `athlete/shin_tracker.yaml` when the athlete reports it
- do not edit generated `athlete/status_dashboard.md` manually unless correcting generated context is explicitly required
