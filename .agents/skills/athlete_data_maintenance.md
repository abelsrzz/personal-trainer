# Skill: Athlete Data Maintenance

## Purpose

Keep athlete data structured, current and useful for planning.

## Update Targets

- `athlete/profile.yaml`
- `athlete/health.yaml`
- `athlete/zones.yaml`
- `athlete/shoes.yaml`
- `athlete/preferences.yaml`
- `athlete/history/`

## Rules

- preserve history
- avoid deleting evidence
- update structured YAML first, then supporting narrative files if useful
