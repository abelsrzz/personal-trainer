# Skill: Running Planning

## Purpose

Turn long-term goals into blocks and weeks.

## Core Principles

- consistency first
- shin safety first
- use heart rate for easy control
- use pace mainly for quality work
- do not fake threshold precision if threshold is unknown
- treat the 35:00 goal as aspirational until checkpoints support it
- use `planning/goal_gates.yaml` before allowing 35:00-specific prescriptions
- use `planning/coach_decision.md` before changing weekly load

## Weekly Planning Checklist

1. identify active block
2. read latest week and reviews
3. refresh or read `athlete/status_dashboard.md` and `planning/coach_decision.md`
4. inspect recent Garmin or manual activity data
5. assess pain, fatigue, compliance and `athlete/shin_tracker.yaml`
6. build the next week accordingly
7. if `planning/weeks/semana_actual.md` is created or modified, generate the PDF and send it by Telegram
